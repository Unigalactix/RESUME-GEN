TOON vs JSON

Created on 2025-11-21 17:31

Published on ---

P.S. If you're building with LLMs, remember that context is currency.

The foundational idea behind the Token-Oriented Object Notation (TOON) is "Sparse Trees, Dense Lists". This format achieves its efficiency by combining YAML-like indentation for nested objects (the "sparse trees") with a compact, CSV-style tabular layout for uniform arrays (the "dense lists").

This design helps eliminate the repetitive keys and braces of JSON, leading to a token reduction of approximately 30–60% for uniform data sets, which directly cuts API costs in LLM and RAG workflows.

You can integrate TOON as the communication layer between your code and the LLM:

• 🐍 Python Users: Install the community-driven Python implementation, which is currently in Beta status (v0.9.x) and working toward full spec compliance.

• 🌐 TypeScript Users: The format has spec-driven implementations available in languages including TypeScript.

• Quick Test: You can convert JSON to TOON to visualize how much token "fat" you can trim from your prompts.
